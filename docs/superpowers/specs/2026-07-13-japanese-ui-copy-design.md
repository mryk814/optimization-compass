# 日本人ユーザー向け UI 文言改善

## 目的

Optimization Compass のブラウザ UI で、内部用の英語コードや専門用語がそのまま表示されている箇所を、日本語ユーザーが意味を推測しやすい表現にする。

研究・実務ユーザーが専門語を確認できるよう、完全な日本語置換ではなく「日本語ラベル（英語の概念名）」を基本形とする。

## 対象範囲

- 対象は `src/optimization_compass/web.py` が返すブラウザ UI の文言。
- API の回答値、SQLite の canonical ID、推薦エンジンの判定ロジックは変更しない。
- CLI の JSON 出力やデータセットのスキーマ変更は今回の対象外。

## UI 仕様

### 質問と選択肢

- 選択肢の `value` には既存の canonical ID をそのまま使う。
- 画面上のラベルには日本語名と英語の概念名を表示する。
- 例:
  - `continuous` → `連続値（continuous）`
  - `integer` → `整数値（integer）`
  - `binary` → `0/1の二値（binary）`
  - `explicit_algebraic` → `数式で表せる（explicit algebraic）`
  - `unknown` → `わからない（unknown）`
- 全選択肢を明示的な表示名マップで管理し、snake_case の機械的変換には依存しない。
- 表示名が未登録の場合は canonical ID をそのまま表示し、回答送信は継続できる。

### 質問の補足

- `why_asked` の前に「この質問で確認すること：」を付け、補足の役割を明示する。
- `beginner_wording` が利用可能な場合は質問本文に使用し、未設定時は現在の `question` を使用する。

### 結果画面

- `支持規則 N` → `一致した判断ルール N件`
- `代表実装` → `主な実装例`
- `docs` → `公式docs`
- `追加確認` → `追加で確認したいこと`
- `発火した規則` → `適用された判断ルール`
- 推薦区分の意味（代替解法、第一候補、条件付き候補、避ける候補）は維持する。

## 実装境界

表示名マップと表示用ヘルパーはブラウザ UI のスクリプト内に置く。canonical ID を API に送る処理と、推薦結果の JSON 構造は変更しないため、既存の API 利用者・テストとの互換性を維持できる。

未登録ラベルのフォールバックは表示上の安全網として一つだけ設ける。旧 UI と新 UI を並存させる分岐や、別バージョンの API は作らない。

## 検証方針

- 既存テストを実行し、推薦結果と API の回答値が変わらないことを確認する。
- `/` の HTML が 200 を返し、質問選択肢の送信値が canonical ID のままであることを確認する。
- 起動中のブラウザ UI で、英語コードだけの選択肢が日本語ラベル（英語併記）に変わり、結果画面の見出し・補助ラベルも意図した文言になることを確認する。
- `verify-data` を実行し、データセット検証結果に変化がないことを確認する。
