# コンテンツ読解原則

Optimization Compass の教材は、情報を増やすことではなく、読者が「いま何を理解すればよいか」を迷わず追えることを優先します。
既存ページの推敲と、これから追加するページは、次の原則を基準にします。

## 1. 最初の視線を固定する

ページを開いた直後に見える主役は、次の3つです。

1. ページタイトル
2. そのページで扱う問いを一文で示す summary
3. `30秒でつかむ` または概念ページの直感的な導入

Page guide、目次、ID、Last reviewed、根拠、関連リンクは重要ですが、本文を読む前の主役にはしません。補助情報は折りたたみ、薄い文字、小さい文字を重ねて本文の視線を奪わないようにします。

## 2. 本文は狭く、十分な大きさで組む

- 日本語本文はおおむね1行35〜55文字を目安にします。
- 本文は `1rem` 前後、行間は `1.8` 前後を基準にします。
- 見出しは本文との差を明確にし、見出しだけを拾っても読解順が分かるようにします。
- 目次、脚注、更新日、ID、図のキャプションは本文より一段下げますが、読めない大きさにはしません。
- 表やコードは横幅を使ってよい領域ですが、前後に「何を見るものか」を説明する文章を置きます。

情報量が多いページを、文字を小さくして一画面に詰め込みません。主線を狭く保ち、補助情報を下位のまとまりへ送ります。

## 3. 読む順番を揃える

### 手法ページ

原則として、次の順番にします。

1. summary
2. `30秒でつかむ`
3. 仕組み・一手の意味
4. まず確認する条件
5. 向いている条件、避ける／切り替える条件
6. `Python`
7. `診断値`
8. `失敗・切替の兆候`
9. `次に読む`

family の選び分け記事は、既存の `30秒でつかむ → まず確認すること → 条件付きの選び分け → うまくいったサインと切替サイン → 小さな比較の型 → コラム → 次に読む` を維持します。

### 概念ページ

原則として、直感、定義、最適化での意味、注意、次に読むの順にします。数式は、式が何を表すかを先に説明します。

## 4. 一文一義、日本語を主言語にする

- 一文では一つの判断だけを伝えます。90字を超える文、読点が3つ以上ある文は分割を検討します。
- 日本語を文の主言語にし、英語の正式用語は初出で日本語の後ろに括弧で示します。例: 制約（constraint）。
- `costを最小化する` のような途中だけ英語になる表現は避け、`コスト（cost）を最小化する` と書きます。
- `step`、`gradient norm`、`solver` などの正準用語は英語のまま保ちますが、用語を並べるだけにせず、何を観測・更新するかを日本語で説明します。
- 技術的事実、数値、主張の強さ、保証の範囲は、読みやすさのために変更しません。

## 5. 補助情報は段階的に見せる

本文の理解に必須でない情報は、本文の主線の後ろに置きます。

- Page guide、目次、構造化データ、トラブルシューティングは開閉可能にします。
- Learning path は「前提」と「次に見る」を主役にし、関連・対比・実装・根拠は本文の読了後にたどれる位置へ置きます。
- 関連リンクは、識別子だけでなく日本語の役割を表示します。
- 同じ意味の説明を、本文、カード、補助パネルで重複させません。

## 6. ページごとの確認項目

各ページを修正するときは、次を順に確認します。

1. 最初の一文だけでページの対象と限界が分かるか。
2. 見出しだけを追って、読む順番が分かるか。
3. 最初の画面に情報が多すぎないか。主役はタイトル・summary・導入に絞られているか。
4. 一文が長すぎないか。日本語と英語の混在が読みを止めていないか。
5. 表、コード、図に入る前に、見る目的が説明されているか。
6. どの診断値を見れば、継続・停止・切替を判断できるか。
7. 内容を変えずに、見出し・順序・余白・文字サイズで読みやすくできているか。

## 7. 文章に呼吸をつくる

説明文の緩急は、装飾や演出ではなく、読者が対象を観察し、判断を保留し、条件を確定し、もう一度診断する順番として設計します。

- 短い文で観測点を置き、長めの文で仕組みや条件を説明し、短い文で判断を止めます。
- 逡巡や反問は、対象に実在する不確実性、トレードオフ、失敗の兆候がある場合だけ使います。教材の情報にないドラマは足しません。
- 箇条書きや表のあとには、列挙した情報がどの判断に使われるかを一文で着地させます。
- 節末の「次は〜を見ます」のような進行実況は避けます。必要な橋渡しは、次の節の冒頭で対象側の問いや違和感として置きます。
- 「本稿では〜を扱います」「要するに、このページの主題は〜です」のように、文書の構成だけを説明する文は削ります。対象の性質、読者が誤りやすい判断、具体的な診断へ書き換えられる場合だけ残します。
- 文章を短くする前に、範囲、比較軸、条件、未確定事項が共有されているかを確認します。短文化による情報の欠落は、読みやすさではありません。
- 結びは抽象的な一般論で閉じず、本文で扱った診断値、失敗条件、次に比較すべき選択へ戻します。
