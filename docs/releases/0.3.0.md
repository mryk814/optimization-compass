# Dataset release 0.3.0

- Release date: 2026-07-15
- Base dataset: 0.2.0
- Runtime SQLite SHA-256: `6fd8851e805ebd4b396905ca33d50b7dd292bfba7a17c02a5d3726e18f3886cc`
- Staged release tree SHA-256: `19683266458b473cb3e23cb4689c40853d4618b62ad61f448f8998035ce79d50`

## Changes from 0.2.0

- Added the seven normalized Atlas metadata tables and CHK013–CHK020 release checks.
- Added deterministic ViewSpec, recommendation SiteData, content, gallery, comparison, and Trace
  assets to the versioned release tree.
- Added `release.json` so dataset version, release date, and runtime database hash are
  machine-readable from the static site.
- Replaced UI/export validator version constants with generated release identity plumbing.
- Replaced fixed content/case ID assertions with minimum policy, unique-ID checks, schema checks,
  and database-backed reference validation.
- Extended atomic publish and failure rollback to `site/public/data`.
- Added code/data/content license declarations, bundled notices, and deterministic CSV ZIP
  `LICENSES/` entries to the v0.3.0 release tree.

## Compatibility

The knowledge schema change is additive. Existing recommendation API and CLI request/response
contracts remain compatible; `dataset_version` changes from `0.2.0` to `0.3.0`. Persisted
AtlasState must treat a different dataset version as an explicit migration boundary rather than
silently rewriting the stored version. The legacy v0.2.0 distribution remains available as the
pinned rebuild base.

## Distribution

The versioned manifest lists filenames, byte counts, and SHA-256 values for SQLite, schema SQL,
JSON, JSONL, CSV directory/ZIP, XLSX, report, release identity, and static site JSON. The Git tag
and GitHub Release are created only after main CI, Pages artifact validation, and post-deploy smoke
checks succeed.
