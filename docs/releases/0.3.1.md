# Dataset release 0.3.1

Release date: 2026-07-15

## Added

- Added canonical learning coverage expectations and five scored priority slices through the
  additive `004_learning_coverage.sql` migration.
- Added complete method/problem/feature-family artifact inventory exports in `coverage.json` and
  the matching `coverage.md` maintainer report.
- Added the public `#/coverage` dashboard, explicit four-state coverage contract, integrity issue
  reporting, and explicit before/after release delta tooling.
- Added CHK021 to validate coverage subject, source, priority, and slice closure.

## Compatibility

This release is additive. Dataset v0.3.0 and its published SHA remain immutable. Consumers that
validate the manifest exactly must accept the new `coverage` asset and dataset version `0.3.1`.
No coverage percentage or implicit v0.2/v0.3.0 baseline is provided.

`release-authority.json.base_dataset_version` remains `0.2.0` by design: it identifies the
physical immutable rebuild base, not the logical predecessor release. The deterministic release
pipeline replays migrations 003 and 004 from that pinned base. Dataset v0.3.0 is the logical
predecessor and remains an immutable published output.
