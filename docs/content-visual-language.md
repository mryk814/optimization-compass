# 教材図の Visual Language

教材図は、文章だけでは読者が頭の中で復元しなければならない空間・更新・時間的関係を先に見せるための補助線です。装飾や性能の印象付けには使いません。

## 追加する判断

次のいずれかが本文の理解の核なら、図を検討します。

- 解・候補・制約が存在する空間や境界を読む。
- 1回の更新で、何が動き、何が保たれ、何が破れるかを追う。
- 同じ語として混同しやすい二つの対象を区別する。
- 時間順の因果が理解対象である。

分類、診断値の一覧、適用条件、注意書きは、図が判断を改善しない限り表または文章で保ちます。

## `atlas-geometric-v1`

生成画像を使う場合は、次の共通言語を守ります。

- 地は暖かい白、構造線は濃紺、可行・観測は青緑、候補・更新は橙、違反だけを赤で示す。
- 平面的で半抽象的な科学教材図にする。写真、人物、製品、擬似3D、光沢、装飾アイコンは使わない。
- 余白を十分に取り、1枚につき主役の関係を一つだけ示す。
- 画像内には文字、数式、軸、凡例、性能値を入れない。正確な用語と数式は本文、SVG、captionで示す。
- 実測値、benchmark、一般的な優劣、収束保証に見える表現を避け、教育用の模式図であることをcaptionで明記する。

## 記事内での位置と密度

- 図は記事冒頭の飾りにはせず、対応する直感・更新・混同の説明の直後に置く。
- 原則は1記事につき主図1枚です。二枚目は、仕組みと失敗条件のように別の問いを解く場合だけ追加します。
- 図の直前に何を見るかを一文で書き、図の直後のcaptionで、読めることと読めないことを一文で固定します。

## 生成画像と正確な図の境界

生成画像は直感を担います。変数名、数式、厳密な矢印、軸、定量値、実行結果は生成画像に委ねません。厳密な対応が教材の主題なら、SVGまたは既存のTheater / Visualization Scenarioを使います。

動的な可視化は、候補・受理判断・分岐・時間変化そのものを追う必要がある場合だけ追加します。scenarioの目的、固定条件、観測量、text alternative、限界は既存のVisualization Scenario contractに従います。

## 図の制作brief

各図は作成前に、少なくとも次を決めます。

```yaml
visual_role: intuition | mechanism | contrast | diagnostic
question: 図を見た直後に答えられる問い
must_show: [主役の関係]
must_not_imply: [性能、保証、実測値など]
style: atlas-geometric-v1
```

本文へ置く図は、具体的なalt textとvisible captionを必須にします。画像単独で理解を完結させず、本文の主張を短縮するためにも使いません。
