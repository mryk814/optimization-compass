# Optimization Compass 0.2.0 release notes

## Breaking browser UI change

Optimization Atlas is now the canonical browser experience. The dependency-free interactive
diagnosis form formerly served from the FastAPI root was supported through 0.1.x and is no longer
part of the application.

Opening `http://127.0.0.1:8000/` now shows a migration landing page with links to Atlas, OpenAPI,
the OpenAPI JSON document, and the health check.

### Migration

- Browser diagnosis: use the published Optimization Atlas or run
  `cd site && npm ci && npm run dev`.
- REST integrations: continue using the existing `/v1/*` endpoints; their contracts are unchanged.
- CLI integrations: `questions`, `recommend`, `verify-data`, `export-site-data`, and `serve` remain
  available.

The support decision and feature matrix are recorded in
[`ADR 0003`](../adr/0003-canonical-browser-experience.md).
