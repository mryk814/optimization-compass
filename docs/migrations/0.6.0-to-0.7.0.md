# Dataset migration 0.6.0 → 0.7.0

Version 0.7.0 adds two append-only authorities.

- `implementation_claims` stores volatile implementation facts as dated claims. Unknown is a
  value state, never an absent row. An active claim is unique by subject and predicate; an old
  claim remains addressable through `valid_from`, `valid_to`, and `replaced_by`.
- `benchmark_contexts` stores the problem instance, hardware/runtime, budgets, tolerance,
  initialization, tuning policy, implementation versions, metrics, and status mapping required
  to interpret a comparison. `comparison_sets.benchmark_context_id` is the canonical link.

Consumers must not derive a performance ranking when the comparison has no complete benchmark
context. Existing implementation columns remain source inputs for this migration, but generated
clients should consume the claim ledger for time-dependent facts.

The initial high-usage cohort is defined from implementations directly exposed by the published
Gallery (`I_ORTOOLS_CPSAT`, `I_OPTUNA`, and `I_CVXPY`), rather than counting internal method-map
fan-out. All three have a verified release claim; OR-Tools 9.15 was verified against the official
Google repository release dated 2026-01-12.
