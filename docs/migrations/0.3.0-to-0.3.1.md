# Migrating dataset 0.3.0 to 0.3.1

Dataset 0.3.1 adds `learning_coverage_expectations` and `learning_slice_priorities`. Apply
`data/migrations/004_learning_coverage.sql`, then load the matching seed collections. The
published v0.3.0 artifacts must not be overwritten.

The repository release builder does not mutate the v0.3.0 SQLite file. For reproducibility it
replays migrations 003 and 004 from the authority's immutable physical v0.2.0 base; v0.3.0 is the
logical predecessor. Existing external v0.3.0 databases may apply migration 004 additively.

Static-site consumers must replace the complete data directory so `release.json`,
`manifest.json`, `coverage.json`, `coverage.md`, and every existing asset all identify v0.3.1.
The manifest parser is exact and now requires:

```json
{"coverage":{"version":"1.0.0","path":"coverage.json","report_path":"coverage.md"}}
```

Release-note comparisons require explicit `--before` and `--after` coverage snapshots.
