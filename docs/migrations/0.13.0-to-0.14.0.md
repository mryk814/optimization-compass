# Dataset migration 0.13.0 → 0.14.0

No table, artifact-envelope, renderer, or frontend contract version changes in this release.

Gallery case `budget-allocation` now references canonical archetype `PA032` and connects to
`PROBLEM_BINARY_KNAPSACK`, `INSTANCE_BINARY_KNAPSACK_4`, both existing search-tree scenarios, and
canonical comparison `COMPARE_KNAPSACK_BNB_BUDGET`. Consumers should no longer interpret this case
as continuous unconstrained problem `PA006`.

Search-tree generator `educational.branch_bound.knapsack.v1` advances from 1.0.0 to 1.1.0. Both
members declare a common `evaluation_budget=9`; the complete member uses `max_nodes=9`, while the
budget-stop member uses `max_nodes=4`. Scenario export now preserves those actual stopping records
instead of replacing the budget-stop limit with the common evaluation allowance. AlgorithmTrace,
SearchTree, renderer, and VisualizationScenario contract versions remain unchanged.

`BENCH_KNAPSACK_BNB_EDUCATIONAL_9` records the exact educational context. Comparison generation
cross-validates the instance, common budget, seed, initial state, generator identity,
implementation-mapping state, and each member's `node_stop_limit`. The comparison is
`contrast_only`, explicitly forbids ranking, and must not be read as a CP-SAT, MIP-solver, or
wall-clock benchmark.

As in 0.13.0, the complete distribution is prepared as a deterministic external bundle under ADR
0014. Git retains only compact release metadata, the current runtime SQLite and version file, and
Pages data after a separate atomic publish step.
