# Migrating dataset 0.3.1 to 0.3.2

Dataset 0.3.2 repairs the canonical relationship between visualization/comparison metadata and
the published learning artifacts. `coverage.json.integrity_issues` no longer reports unresolved
canonical scenario or comparison IDs.

The canonical first-order comparison is now `COMPARE_GRADIENT_FAMILY`, matching its quadratic
objective, initial point, budget, and member parameters. The retired
`COMPARE_FIRST_ORDER_ROSENBROCK` route remains an alias of that comparison so existing links keep
opening the same public lesson.

Every visualization scenario now states one identity status:

- `canonical` — the scenario directly corresponds to a canonical database record;
- `derived` — the artifact is a documented variant of a canonical scenario; or
- `generated_only` — the generated educational artifact is intentionally not a database scenario.

Static-site consumers must replace the whole data directory. The manifest, release identity,
scenario index, comparisons, coverage report, and all trace payloads identify v0.3.2 together.
