# Migrating dataset 0.4.0 to 0.5.0

Dataset 0.5.0 replaces `demo_objectives` with canonical problem definitions and
instances.

## Schema changes

- Added `problem_definitions`, `problem_definition_archetypes`,
  `problem_definition_features`, and `problem_instances`.
- Recreated `demo_scenarios` and `comparison_sets` with `problem_instance_id` foreign
  keys.
- Removed `demo_objectives`; there is no parallel compatibility table.

## Site contract

`manifest.json` now includes the versioned `problems.json` asset. Existing
`AlgorithmTrace.objective_id` values remain stable and identify a canonical problem
instance.

Consumers that queried `demo_objectives` should join `problem_instances` to
`problem_definitions`. Display ranges and known references come from
`problem_instances.display_json` and `known_reference_json`.
