# Dataset migration 0.7.0 → 0.8.0

Version 0.8.0 normalizes the first twelve high-value failure modes without duplicating their
canonical `FMxxx` identity. New child tables describe triggers, observable or explicitly
non-visual symptoms, diagnostics, conditional mitigations, affected entities, scope, severity,
recoverability, confidence, and failure-contrast scenarios.

Consumers should use `failure-modes.json` or the normalized tables. The old prose columns remain
the long description attached to the same canonical row; they are not a fallback execution path.
Implementation-specific failures must retain `implementation_only` specificity and must not be
presented as a general property of the mathematical method.
