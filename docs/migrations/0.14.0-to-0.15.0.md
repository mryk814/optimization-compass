# Dataset migration 0.14.0 → 0.15.0

No table, artifact-envelope, renderer, or frontend contract version changes in this release.

Gallery case `EC017` now distinguishes its real cost-performance formulation from the fixed
educational problem `INSTANCE_BIOBJECTIVE_QUADRATIC_2D`. The educational instance uses
`0 ≤ x,y ≤ 2` and minimizes `f₁=x²+y²` together with `f₂=(x−2)²+(y−2)²`; consumers must not read
those dimensionless variables as the Case's literal design dimensions or operating conditions.

Derived scenario `SCENARIO_BIOBJECTIVE_PREFERENCE_SENSITIVITY` reuses the existing
`biobjective-quadratic-pareto-front` artifact and names `M_WEIGHTED_SUM` as the preference-selection
method. Primary scenario `SCENARIO_BIOBJECTIVE_QUADRATIC` remains the front-generation lesson.

`COMPARE_PARETO_PREFERENCE` now contains three non-ranking members with `weight_f1` values 0.2,
0.5, and 0.8. All members share the same 81 sampled points and select only canonical results already
recorded in the artifact. Browser clients must reject a missing, duplicate, or mismatched preference
selection instead of recomputing a substitute.

The Pareto renderer reports sampled, dominated, non-dominated, and analytic-reference counts as
different quantities. Neither the 25 sampled non-dominated points nor the 21 analytic reference
samples are an algorithmic coverage certificate.

The learning-journey report advances from four complete and seven partial journeys to five complete
and six partial journeys. Contract versions remain unchanged.

As in 0.14.0, the complete distribution is prepared as a deterministic external bundle under ADR
0014. Git retains only compact release metadata, the current runtime SQLite and version file, and
Pages data after a separate atomic publish step.
