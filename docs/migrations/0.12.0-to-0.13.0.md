# Dataset migration 0.12.0 → 0.13.0

No table, artifact-envelope, renderer, or frontend contract version changes in this release.

The canonical benchmark-context builder adds `BENCH_BO_EDUCATIONAL_10` for the fixed
one-dimensional expensive black-box lesson. Unlike the general `BENCH_BO` context, this
context records the actual 10-oracle-evaluation budget, seed `2604`, initial design
`[-2.6, 0, 2.6]`, deterministic educational generator, and an explicit
`implementation_mapping_status=not_applicable`. Comparison export cross-validates those
fields against every referenced scenario and run.

Gallery case `hyperparameter-search` is corrected to archetype `PA039` and connected to the
existing four Bayesian-optimization scenarios, the existing executable problem instance,
and the new canonical comparison `COMPARE_BO_ACQUISITION_NOISE_BASELINE`. The comparison
keeps the problem, initial design, seed, and 10-evaluation budget fixed while changing the
Expected Improvement exploration coefficient, observation noise, or proposal policy. It is
`contrast_only`, forbids ranking, and treats acquisition coefficient as `not_applicable` for
the uniform-random baseline.

Bayesian Optimization Theater scenario identity now comes from the canonical path segment and
is resolved by exact scenario ID. Unknown IDs return Not Found instead of selecting a preset by
substring. Consumers caching Gallery, comparison, scenario, journey, entity-link, content,
search, or retrieval indexes should refresh them together.

Starting with 0.13.0, the complete versioned distribution is prepared as a deterministic
external bundle under the retention contract in ADR 0014. Git retains only the current runtime
SQLite, `DATASET_VERSION`, Pages data, compact release metadata, and release catalog after the
separate atomic publish step.
