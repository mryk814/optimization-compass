# Migrating dataset 0.3.2 to 0.4.0

Dataset 0.4.0 adds the canonical atomic-predicate authority used by recommendation,
comparison eligibility, Map details, and method pages.

1. Apply `data/migrations/005_atomic_predicates.sql` after migrations 003 and 004.
2. Load `data/seeds/atomic_predicates.json` as a single validated `PredicateCatalog`.
3. Treat `predicate_coverage.status = complete` as the only state that permits enforcement.
4. Remove each `decision_rule_target_retirements` target from the legacy rule authority and
   compile it back only after the linked policy excludes the legacy rule condition.
5. Consume SiteData `2.0.0`; version 1 clients must reject the new exact-key payload.

The first migration contains 15 representative methods. Six have complete coverage for their
currently enforced incompatibility rules; nine remain explicitly partial and are display-only.
