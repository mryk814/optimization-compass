# Dataset migration 0.10.0 → 0.11.0

No table is added or removed.

The migration adds one canonical method, one primary source, and one method hierarchy edge. The existing SciPy Trust Region Reflective implementation is updated and reconnected from the broad `M_GAUSS_NEWTON` / `M_TRUST_KRYLOV` mappings to `M_TRUST_REGION_REFLECTIVE`.

The generic versioned `important_option_defaults` claim now carries API-specific default behavior for `scipy.optimize.least_squares` and bounds-dependent `scipy.optimize.curve_fit`. This is implementation metadata and does not change recommendation rules.

Consumers that cached the previous method mapping should resolve the generated `method_implementation_map` and entity-link artifacts again rather than retaining the 0.10.0 fan-out.
