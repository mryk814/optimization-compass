# Dataset migration 0.15.0 → 0.15.1

This is a corrective metadata release. There are no table, artifact-envelope, renderer, or frontend
contract version changes.

In 0.15.0, derived scenario `SCENARIO_BIOBJECTIVE_PREFERENCE_SENSITIVITY` reused the canonical
Pareto artifact but labelled its run as `M_WEIGHTED_SUM` while retaining the NSGA-II visualization
profile. That mixed the identity of the front-generation run with the later preference-selection
operation.

In 0.15.1, the derived scenario shares the primary scenario's canonical
`RUN_BIOBJECTIVE_GRID / M_NSGA_II / PROFILE_NSGA_II_PARETO_FRONT` run unchanged. The three
`COMPARE_PARETO_PREFERENCE` members continue to use `M_WEIGHTED_SUM`; each member's `weight_f1`
selects one unique, already-recorded result from the shared artifact. Browser clients continue to
reject missing, duplicate, or mismatched selections rather than synthesizing a substitute.

Dataset 0.15.0 remains an immutable historical release and is superseded by 0.15.1.
