# Dataset migration 0.11.0 → 0.12.0

No table, artifact, renderer, or scenario-envelope version is added or removed.

The versioned problem-suite seed adds one nonlinear least-squares definition and
one executable three-parameter exponential-decay instance. The released SQLite
database, runtime resource, problem catalog, and site exports are regenerated from
that seed as one atomic release.

The Gallery, scenario, and comparison inputs add the `EC013` learning journey.
Its three normal-initialization members intentionally share one identical,
solver-independent damped Gauss-Newton diagnostic-probe history. The
poor-initialization member applies the same rule from a different initial point.
All four traces reuse `AlgorithmTrace` 1.0 and renderer contract 1.0.0.
`residual_vector` and `jacobian` are additive oracle-policy vocabulary; residual
norm and Jacobian rank remain ordinary trace metrics rather than new payload
fields.

Consumers that cache problem, scenario, comparison, journey, entity-link, search,
or retrieval indexes should refresh them together. No member solver is executed;
the member names are applicability lenses only. The traces must not be interpreted
as SciPy/Ceres internal iterations, attained outcomes, convergence speed, or solver
rankings.
