# Migrating dataset 0.5.1 to 0.6.0

Dataset 0.6.0 makes semantic Views canonical dataset resources instead of a
single exporter-specific tree.

- `view_presets.view_id` is the stable public ID used by site URLs and state.
- `filter_policy_json` is the only authority for authored View groups and their
  canonical question, feature, method, and alternative members.
- `limitations_ja` / `limitations_en` state what each View does not establish.
- `focus_fallback_entity_types_json` defines deterministic focus restoration
  when a selected entity is absent from another View.
- The site manifest now publishes four semantic View artifacts: problem
  structure, Oracle, Guarantee, and method mechanism.

Consumers must select Views from `manifest.json`; do not infer paths or rebuild
trees independently from the raw database.
