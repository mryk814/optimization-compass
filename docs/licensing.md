# Licensing decision

公開前に**コード**と**データベース内容**を分けて判断してください。

## Code candidates

- MIT: 短く、再利用しやすい
- Apache-2.0: 明示的な特許条項を含む

## Data candidates

- CC BY 4.0: 帰属を条件に幅広く再利用
- ODbL 1.0: データベースの共有継承を重視

ただし、引用・要約・URL・製品情報など各情報源の権利と利用条件を確認する必要があります。データライセンスを付けても、第三者コンテンツそのものの権利を再許諾できるとは限りません。

## Recommended preparation

1. `sources` をsource type別に棚卸し
2. 長い転載がなく、事実・短い要約・識別情報中心であることを確認
3. contributor agreement または DCO の要否を決定
4. code license と data license をREADMEで明確に分離
