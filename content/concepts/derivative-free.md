微分を使わない最適化（derivative-free optimization）は、目的関数や制約の評価値を比べ、勾配を直接入力せずに候補点を更新する考え方です。

## 直感

勾配がないときも、候補点を評価して「どちらがよいか」は比べられます。
微分を使わない方法は、その比較から次の候補点を組み立てます。

対象は数式として微分できない関数だけではありません。
解析勾配を実装できない場合や、勾配がノイズに埋もれる場合も含みます。
勾配一回の取得が関数評価より大幅に高い場合も候補です。

> 「微分を使わない」は「情報を使わない」という意味ではありません。探索履歴、点の配置、評価値の差を更新へ使います。

## 何が「微分なし」なのか

solverへ勾配を渡さないことと、問題に滑らかさがないことは別です。
滑らかな目的でも、値だけを返す評価器なら微分なしの探索を選べます。
逆に、数値差分を内部で計算するsolverは利用者が勾配を渡さなくても評価回数を使います。

選択前に、評価器が返せる情報を分けます。

| 利用できる情報 | 先に確認すること |
| --- | --- |
| 目的値だけ | 同点やノイズをどう比較するか |
| 目的値と制約値 | infeasibleな候補をどう扱うか |
| 複数回の再評価 | ばらつきをどの単位で集約するか |
| 失敗status | 失敗と大きな目的値を区別できるか |

## 探索の役割を分ける

局所探索は現在点の近くを詳しく調べます。
Nelder–MeadやPowell法はSciPyの公式APIでも微分を使わない局所法として選べます。
初期点の近くにある改善を探すため、大域最適性を自動では保証しません。

大域探索は離れた領域にも候補を配ります。
ただし、範囲を広げるほど同じ予算で一領域を調べる密度は下がります。
高価な評価では、既評価点を使うsurrogateで次の評価点を絞る方法もあります。

これらは「微分なし」という共通点だけで同じsolverではありません。
局所性と変数型を分けて確認します。
制約の扱いと並列性も選択条件です。

## 評価budgetを先に決める

微分を使わない探索では、評価回数が主な資源になります。
一回の評価時間と同時実行数を測り、停止までの上限を先に置きます。
ノイズがある場合は、再評価に使う回数も同じbudgetへ含めます。

比較実験では、solverの反復回数だけを揃えてはいけません。
一反復で必要な評価回数が方法ごとに違うためです。
目的評価、制約評価、失敗した評価を同じ規則で数えます。

::: warning
評価失敗を一律に大きな目的値へ置き換えると、悪い候補と実行不能な候補が混ざります。失敗statusを別に保存してください。
:::

## 診断値

| 診断値 | 読み方 |
| --- | --- |
| best-so-far objective | 予算の追加で最良値が更新されているか |
| 評価回数と経過時間 | budgetをどこで消費したか |
| 候補点の距離 | 同じ狭い領域だけを繰り返していないか |
| 再評価のばらつき | 改善量がノイズより大きいか |
| infeasible率と失敗率 | 探索範囲や制約処理が適切か |

最良値だけでなく、評価履歴を残します。
同じ点の再評価や失敗候補が多い場合、solverの停止理由だけでは原因を判別できません。

## 失敗・切替の兆候

- 評価budgetを使っても最良値が更新されない場合は、初期点と探索範囲を見直します。
- 候補の多くがinfeasibleなら、制約を扱える方法か変数表現へ切り替えます。
- 改善量が再評価のばらつき以下なら、再評価規則かノイズを扱う方法を検討します。
- 局所探索で初期点ごとの結果が大きく違うなら、多点開始か大域探索を比較します。
- 変数次元に対して評価budgetが小さいなら、変数を減らすか事前知識を探索へ入れます。
